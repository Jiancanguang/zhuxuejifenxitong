## 1. 系统概述

**班级宠物园** 是一套课堂积分管理工具，老师通过日常加减分来奖惩学生行为，积分会转化为"宠物食物"，驱动每位学生的虚拟宠物从蛋孵化、成长直到毕业。毕业后可获得永久徽章，徽章还能在班级商店中兑换实际奖品。

### 技术栈

| 层级 | 技术 |
|------|------|
| 前端 | React + TypeScript + Tailwind CSS + Vite |
| 后端 | Express.js (Node.js) |
| 数据库 | SQLite (本地) |
| 实时通信 | Server-Sent Events (SSE) |
| 导出 | jsPDF + Canvas (证书/贴纸) |

## 2. 核心概念

### 2.1 积分（食物）

学生通过课堂表现获得积分，积分在系统内以"食物"形式呈现。食物直接驱动宠物成长。

### 2.2 宠物

每位学生拥有一只虚拟宠物，从50多种萌宠中选择。宠物随着食物积累共经历 **10个成长阶段**，从蛋到满级形态。

### 2.3 毕业

宠物达到满级（第10阶段）后可以"毕业"，意味着该宠物养成完毕。毕业后学生获得一枚徽章，宠物重置为蛋，可以选择新宠物继续养成。

### 2.4 徽章

每次毕业获得一枚徽章，记录了毕业宠物的品种和昵称。徽章既是荣誉展示（永久保存在徽章墙），也是商店兑换的"货币"。

### 2.5 兑换

学生可以用徽章在班级商店兑换老师设置的奖品（如免作业卡、小零食券等）。可用余额 = 累计获得徽章数 − 已消耗徽章数。

## 3. 快速上手

### 第一步：注册登录

1. 打开系统，进入登录/注册页面
2. 输入用户名和密码完成注册
3. 登录后进入主界面

### 第二步：创建班级

1. 点击左上角班级管理入口
2. 创建新班级并命名（如"三年级1班"）

### 第三步：添加学生

1. 在班级页面中添加学生（输入学生姓名）
2. 每个学生创建后会显示为一个"神秘蛋"卡片

### 第四步：选择宠物

1. 点击学生卡片上的"选择宠物"按钮
2. 从50多种宠物中挑选一只
3. 宠物开始第1阶段，等待投喂

### 第五步：日常打卡

1. 点击学生卡片进入打卡界面
2. 选择积分项目（如"答对问题 +2"）
3. 学生获得食物，宠物自动成长

### 第六步：毕业与徽章

1. 宠物达到满级后弹出毕业庆祝页面
2. 点击"收集徽章"完成毕业
3. 学生获得徽章，宠物重置，开始新一轮养成

## 4. 功能详解

### 4.1 班级管理

| 操作 | 说明 |
|------|------|
| 创建班级 | 输入班级名称，每位老师可管理多个班级 |
| 切换班级 | 在班级列表中快速切换当前操作的班级 |
| 重命名班级 | 修改已有班级的名称 |
| 删除班级 | 删除班级及其所有学生数据（不可恢复） |
| 复用配置 | 将一个班级的积分项目、奖品、阶段阈值复制到另一个班级 |

### 4.2 学生管理

**添加学生**
- 输入学生姓名即可添加
- 新学生初始状态：无宠物（神秘蛋）、积分为0

**学生卡片显示信息**
- 学生姓名
- 当前宠物及成长阶段图像
- 食物（积分）数量
- 进度条（距离下一阶段的进度）
- 已获得的徽章数量

**排序方式**（4种可选）
- **手动排序**：拖拽调整顺序
- **按姓名排序**：按拼音字母排列
- **排行榜排序**：按积分从高到低
- **按进度排序**：按成长阶段排列

**批量操作**
- 勾选多个学生，批量打卡加分

### 4.3 积分打卡

#### 默认积分项目（14项）

**加分项：**

| 项目 | 分值 | 说明 |
|------|------|------|
| 早读打卡 | +1 | 按时参加早读 |
| 答对问题 | +2 | 正确回答课堂提问 |
| 作业优秀 | +3 | 作业质量优异 |
| 完成背诵 | +2 | 完成背诵任务 |
| 积极举手 | +1 | 课堂上积极举手 |
| 帮助同学 | +2 | 主动帮助同学 |
| 值日认真 | +2 | 认真完成值日工作 |
| 课外阅读 | +1 | 阅读课外书籍 |
| 进步明显 | +3 | 表现有明显进步 |
| 才艺展示 | +3 | 展示个人才艺 |

**扣分项：**

| 项目 | 分值 | 说明 |
|------|------|------|
| 迟到 | −1 | 上课迟到 |
| 玩手机 | −2 | 课堂上使用手机 |
| 打瞌睡 | −1 | 课堂上睡觉 |
| 未交作业 | −2 | 未按时提交作业 |

> 所有积分项目均可在设置中自定义名称、图标和分值。

#### 打卡流程

1. 点击学生卡片 → 弹出积分选择面板
2. 选择对应的积分项 → 确认
3. 系统自动更新食物数量和宠物阶段
4. 卡片上显示投喂动画效果

#### 批量打卡

1. 开启批量选择模式
2. 勾选需要打卡的学生
3. 选择积分项 → 一次性为所有选中学生加分

### 4.4 宠物系统

#### 宠物种类（50+种）

系统提供50余种虚拟宠物供选择，包括但不限于：

- **猫类**：白猫、布偶、橘猫、黑猫、虎斑、加菲猫、金渐层、蓝猫、暹罗猫、银渐层、三花、狸花猫
- **犬类**：柯基、柴犬、法斗、金毛、拉布拉多、萨摩耶、泰迪、西高地、吉娃娃、中华田园犬、雪瑞纳、比熊、边牧、大白熊
- **其他萌宠**：小兔子、松鼠、仓鼠、小猪仔、鸭子、小黄鸡、小羊羔、鹦鹉
- **野生动物**：熊猫、猴子、马、企鹅、老虎、豹子、梅花鹿、大象、狐狸、青蛙、乌龟、鳄鱼、变色龙
- **幻想生物**：九尾狐、恐龙
- **网红动物**：卡皮巴拉

#### 成长阶段

宠物共有 **10个成长阶段**，每个阶段有对应的形态图片。

**默认阶段阈值：**

| 阶段 | 所需食物（累计） | 说明 |
|------|-----------------|------|
| 1 | 0 | 蛋/幼崽（初始） |
| 2 | 5 | 刚出生 |
| 3 | 10 | 幼年 |
| 4 | 20 | 成长中 |
| 5 | 30 | 少年 |
| 6 | 45 | 青年 |
| 7 | 60 | 成年 |
| 8 | 75 | 壮年 |
| 9 | 90 | 强大 |
| 10 | 100 | 满级（可毕业） |

> 阈值可在设置中自定义，调整后会自动重新计算所有学生的阶段。

#### 宠物昵称

- 点击宠物可以为其取昵称
- 昵称会显示在学生卡片和徽章上
- 毕业时昵称会被永久保存到徽章中

### 4.5 徽章系统

#### 获取方式

宠物达到第10阶段（满级）→ 触发毕业流程 → 获得1枚徽章

#### 毕业流程

1. 宠物达到满级时弹出 **毕业庆祝页面**（带烟花、星星飘落等动画效果）
2. 显示满级宠物的完整形态
3. 两个选择：
   - **「收集徽章」**：立即完成毕业，获得徽章
   - **「稍后再说」**：暂不毕业，下次再操作

#### 毕业后的变化

| 项目 | 毕业前 | 毕业后 |
|------|--------|--------|
| 宠物 | 满级形态 | 重置为无（可重新选择） |
| 积分/食物 | 满值 | 清零 |
| 成长阶段 | 第10阶段 | 第1阶段 |
| 徽章 | N枚 | N+1枚 |

#### 徽章墙

点击学生卡片上的徽章图标可打开 **徽章墙**，展示：

- **统计面板**：累计获得 / 已消耗 / 可用余额
- **徽章展示区**：网格排列，每枚显示宠物满级形态图片、昵称和编号
- **点击预览**：放大查看单枚徽章详情（宠物名称、获得时间）
- **兑换记录**：历史兑换商品列表

#### 徽章撤销

如果毕业操作有误，可以在成长记录中撤销：
- 撤销后对应徽章被删除
- 学生恢复到毕业前的状态（积分、宠物、阶段全部恢复）

### 4.6 商店与兑换

#### 默认奖品列表

| 奖品名称 | 徽章花费 | 说明 |
|----------|---------|------|
| 免作业卡 | 3 | 免除一次作业 |
| 前排座位券 | 2 | 自选座位一周 |
| 小零食券 | 1 | 获得一份小零食 |
| 贴纸奖励 | 1 | 获得一张贴纸 |
| 电影时间 | 5 | 观看一段动画视频 |
| 小组长体验 | 4 | 担任一天小组长 |

> 所有奖品均可在设置中自定义名称、花费、图标和颜色。

#### 兑换流程

1. 打开班级商店
2. 浏览可兑换奖品及所需徽章数
3. 确认兑换 → 系统检查可用余额是否充足
4. 兑换成功后更新徽章余额和库存

#### 库存管理

- 每个奖品有独立库存
- 库存为0时无法兑换
- 老师可在设置中随时调整库存数量

### 4.7 排行榜

排行榜提供两种视图：

**个人排行**
- 按学生累计食物数量从高到低排名
- 显示每位学生的宠物头像、姓名、积分

**小组排行**
- 按小组内成员积分总和排名
- 便于小组竞赛使用

### 4.8 分组管理

#### 创建分组

- **手动创建**：逐个创建分组并命名
- **随机分组**：系统自动将学生随机分配到指定数量的组中

#### 分组功能

- 自定义组名和颜色（10种颜色可选）
- 拖拽调整分组顺序
- 批量分配学生到指定组
- 按组筛选显示学生卡片
- 小组排行榜

#### 分组颜色

红色、橙色、黄色、绿色、青色、蓝色、紫色、粉色、棕色、灰色

### 4.9 成长记录

成长记录页面展示学生的完整操作历史：

| 记录类型 | 说明 |
|----------|------|
| `checkin` | 积分打卡（加/减分） |
| `graduate` | 宠物毕业 |
| `redeem` | 商店兑换 |
| `revoke` | 撤销操作 |
| `rename` | 宠物改名 |

#### 记录管理

- 按学生或类型筛选
- 单条撤销：撤销某次打卡或毕业（恢复原状）
- 批量撤销：撤销同一批次的打卡记录
- 删除记录：删除历史条目（不恢复状态）
- 清空记录：清除班级所有历史

> 每个班级最多保存 **10,000条** 历史记录。

### 4.10 证书与贴纸导出

系统支持将学生的毕业徽章导出为可打印的证书或贴纸：

#### 证书导出

- 以精美证书模板展示学生姓名、毕业宠物、获得时间
- 支持导出为 **PNG图片** 或 **PDF文件**

#### 贴纸导出

- 横版布局，展示多枚徽章
- 适合打印后裁剪使用

#### 批量导出

- 选择多个学生一次性导出
- 打包为 ZIP 文件下载

#### 基础设置

| 设置项 | 说明 | 默认值 |
|--------|------|--------|

## 6. 数据说明

### 数据库表结构

| 表名 | 用途 | 关键字段 |
|------|------|----------|
| `users` | 教师账号 | username, password_hash, system_title |
| `classes` | 班级信息 | title, stage_thresholds, rewards, score_items, inventory |
| `students` | 学生数据 | name, pet_id, pet_stage, food_count, pet_nickname |
| `groups` | 分组信息 | name, color_token, sort_order |
| `badges` | 毕业徽章 | pet_id, pet_name, earned_at |
| `history_records` | 操作历史 | type, score_value, cost, badge_id, meta |

### 实时同步机制

系统使用 **Server-Sent Events (SSE)** 实现多设备实时同步：

- 打卡、毕业、兑换、撤销等操作会即时推送到所有已连接的客户端
- 心跳间隔：15秒
- 如果连接断开，客户端会自动重连

### 数据安全

- 用户密码使用 **bcrypt** 加密存储
- 使用 JWT Token 进行身份验证
- 登出或修改密码后旧 Token 自动失效
- 危险操作（重置进度、复用配置、修改密码）需要二次密码确认

## 7. 常见问题

### Q: 学生积分被扣到负数怎么办？

A: 系统会自动将食物数量限制在最低0，不会出现负数。

### Q: 毕业操作失误了怎么办？

A: 在成长记录中找到该条毕业记录，点击"撤销"即可恢复学生之前的宠物、积分和阶段。

### Q: 修改了阶段阈值后，已有学生会受影响吗？

A: 会。系统会根据新的阈值重新计算所有学生的成长阶段，需要密码确认。

### Q: 一个班级最多能有多少条记录？

A: 每个班级最多保存 10,000 条历史记录。

### Q: 多个设备可以同时操作吗？

A: 可以。系统通过实时同步机制确保多设备数据一致性。

### Q: 删除班级后数据能恢复吗？

A: 不能。删除班级会同时删除该班级下的所有学生、徽章和历史记录，请谨慎操作。

### Q: 徽章可以重复使用吗？

A: 徽章数量是固定的（每次毕业获得1枚）。兑换商品会消耗徽章余额，但不会删除徽章本身。可用余额 = 累计获得 − 累计消耗。

### Q: 学生可以同时养多只宠物吗？

A: 不可以。每位学生同时只能养一只宠物，毕业后才能选择下一只。
